# 前后端交互说明

当前仓库里已实现：**学生画像**、**辅导员总览仪表盘**、**群体聚类雷达**；其它辅导员端接口仍由前端 mock，便于按模块逐步接入。

---

## 1. 环境与基址

| 角色 | 说明 |
|------|------|
| 前端开发服务器 | Vite，默认 `http://127.0.0.1:5173` |
| 后端 API | Flask（开发态 `debug=True`），默认 `http://127.0.0.1:8000` |
| 代理 | `frontend/vite.config.ts` 将 `/api` 转发到 `http://127.0.0.1:8000` |

**启动**

```bash
# 后端（在 backend 目录，需已安装 flask、flask-cors）
flask --app api_server.main run --host 127.0.0.1 --port 8000 --debug
# 或: python -m api_server.main

# 前端（在 frontend 目录）
npm run dev
```

Flask 默认不提供 OpenAPI 页面；需要时可自行加 `flask-smorest` 或手写接口说明（即本文档）。

---

## 2. 当前已实现接口

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/counselor/dashboard` | 总览：KPI、`groupDistribution`（行为 K-Means 或占位）、近 7 日预警趋势、预警列表、站外热点 |
| `GET` | `/api/counselor/clusters?method=…` | 群体聚类雷达：`behavior_kmeans` \| `text_topic` \| `temporal`；非法 `method` 返回 **400** |
| `GET` | `/api/profile/{student_id}` | 查询 `student_profiles` 画像；无记录时 **404** |

### 2.1 `GET /api/counselor/dashboard`

**响应体**（与 `CounselorDashboard` 一致，camelCase）：

- `kpis`：`totalStudents`, `warningStudents`, `highRiskStudents`, `inProgressTasks`, `closedTasks`
- `groupDistribution`：`[{ name, value }, ...]`（与总览页默认「行为 K-Means」雷达一致的数据源）
- `warningTrend`：`[{ day, value }, ...]`
- `alerts`：`[{ id, title, risk_level, summary, created_at }, ...]`
- `hot_topics`：`[{ id, platform, title, heat_label, captured_at }, ...]`

**前端入口**：`getCounselorDashboard()`（`counselor.ts`）；请求失败时回退 mock。

---

### 2.2 `GET /api/counselor/clusters`

**Query**

| 参数 | 说明 |
|------|------|
| `method` | 可选，默认 `behavior_kmeans`。`text_topic`、`temporal` 当前为服务端占位分布（与产品演示一致），待接入独立分析流水线后可替换。 |

**响应体**：`[{ "name": string, "value": number }, ...]`

**前端入口**：`getClusterDistribution(method)`（`counselor.ts`）；失败时回退 mock。

---

### 2.3 `GET /api/profile/{student_id}`

**成功响应**（JSON，字段与 `frontend/src/types.ts` 中 `StudentProfile` 一致）：

- `student_id`, `basic_tags`, `behavior_tags`, `cognitive_tags`, `radar_scores`, `intervention_action`, `last_computed_at`, `activity_trend`

**错误**：404，`{"detail":"student_id not found"}`

**前端入口**：`frontend/src/api/counselor.ts` 中的 `fetchStudentProfileFromApi` → `getStudentProfileDetail`（失败则走本地 mock）。
